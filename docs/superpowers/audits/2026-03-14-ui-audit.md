# WhisloAI UI/UX Audit (2026-03-14)

## Anti-Patterns Verdict
Pass with caveats. The interface does not look like generic AI slop, but it has recurring quality gaps (inconsistent tokens, limited focus treatment, small control targets in widget mode, and mixed status behavior) that reduce polish and accessibility.

## Executive Summary
- Total issues: 17
- Critical: 1
- High: 6
- Medium: 7
- Low: 3
- Highest-impact areas: accessibility targets/focus, state feedback consistency, and token normalization

Top priorities:
1. Raise widget and recording controls to 44x44 minimum and ensure semantic controls.
2. Standardize status/focus states across Main, Settings, and Widget.
3. Replace hardcoded color patterns with semantic tokens to improve consistency and maintainability.

## Detailed Findings

### Critical
1. `src/widget.html` + `src/styles.css`
- Severity: Critical
- Category: Accessibility
- Issue: Stop-recording affordance uses non-semantic `span` and target below 44x44.
- Impact: Keyboard/screen-reader reliability risk and difficult hit target.
- Standard: WCAG 2.5.5 (Target Size), semantic control expectations.
- Recommendation: Convert to `<button>` and set min 44x44.
- Suggested command: `harden`

### High
1. `src/styles.css`
- Category: Accessibility
- Issue: Focus styles are fragmented and mostly `:focus`-specific to settings inputs.
- Impact: Keyboard users may miss focus on key buttons in main/widget.
- Recommendation: Introduce global `:focus-visible` treatment.
- Command: `harden`

2. `src/styles.css` (widget controls)
- Category: Responsive/A11y
- Issue: Quick toolbar buttons are 30x30.
- Impact: Difficult usage for touch/precision constraints.
- Recommendation: Raise icon controls to at least 44x44.
- Command: `harden`

3. `src/main.js`
- Category: Theming/Resilience
- Issue: Inline text color assignment (`style.color`) for recording/onboarding statuses.
- Impact: Breaks design system consistency and theming updates.
- Recommendation: Use class/data-tone based styles.
- Command: `normalize`

4. `src/settings.html`
- Category: Theming/Consistency
- Issue: Inline style attributes for spacing in transcription section.
- Impact: Non-reusable and inconsistent with system spacing rhythm.
- Recommendation: Move to reusable CSS utility classes.
- Command: `normalize`

5. `src/styles.css`
- Category: Performance
- Issue: `quick-tool-btn` transitions include layout properties (`width`, `padding`, `border-radius`).
- Impact: Extra layout work during state transitions.
- Recommendation: Restrict to transform/opacity/color transitions.
- Command: `optimize`

6. `src/index.html` + `src/main.js`
- Category: UX/Onboarding
- Issue: Onboarding step progression is text-only and lacks explicit step state markers.
- Impact: Lower clarity during permission troubleshooting.
- Recommendation: Add pending/checking/ready/action-required states.
- Command: `onboard`

### Medium
1. `src/styles.css`
- Category: Theming
- Issue: High number of hardcoded hex colors across surfaces/statuses.
- Impact: Harder long-term consistency and updates.
- Recommendation: Introduce semantic token layer and migrate frequently used colors.
- Command: `normalize`

2. `src/styles.css`
- Category: Motion
- Issue: No `prefers-reduced-motion` fallback.
- Impact: Motion-sensitive users may get unnecessary animation.
- Recommendation: Add reduced-motion media query guardrails.
- Command: `harden`

3. `src/main.js`
- Category: UX Feedback
- Issue: Main status predominantly uses neutral/error; loading/success are underutilized.
- Impact: State transitions less clear than Settings UI.
- Recommendation: Support full tone model (`neutral/loading/success/error`).
- Command: `clarify`

4. `src/styles.css`
- Category: Visual Hierarchy
- Issue: Base spacing/typographic rhythm is good but inconsistent between Main and Settings cards.
- Impact: Perceived cohesion drop between windows.
- Recommendation: Harmonize radii, spacing cadence, and heading rhythm.
- Command: `polish`

5. `src/widget.js`
- Category: Discoverability
- Issue: Busy/recording state is functional but abrupt in visual transition.
- Impact: Users can perceive state jumps as glitches.
- Recommendation: Add short, meaningful transition choreography.
- Command: `animate`

6. `src/index.html`
- Category: Microcopy
- Issue: A few labels can be clearer and more action-oriented.
- Impact: Slight cognitive load for first-run users.
- Recommendation: Simplify and standardize verbs.
- Command: `clarify`

7. `src/settings.html`
- Category: Information Architecture
- Issue: Dense provider form actions with equal visual weight.
- Impact: Primary action can be harder to parse quickly.
- Recommendation: Reinforce primary/secondary/danger hierarchy.
- Command: `polish`

### Low
1. `src/styles.css`
- Category: Composition
- Issue: Some border colors vary by tiny increments without semantic meaning.
- Impact: Minor visual noise.
- Recommendation: Normalize border token usage.
- Command: `normalize`

2. `src/styles.css`
- Category: Motion
- Issue: Recording visualizer animations are useful but could benefit from timing consistency.
- Impact: Minor polish gap.
- Recommendation: Align durations/easing across states.
- Command: `animate`

3. `src/index.html`
- Category: UX Writing
- Issue: “Insert in cursor” can be interpreted ambiguously by some users.
- Impact: Minor comprehension issue.
- Recommendation: Consider “Insert at cursor”.
- Command: `clarify`

## Patterns & Systemic Issues
- Color/theming decisions are repeated per component instead of expressed as reusable tokens.
- Status feedback behavior differs between Main and Settings.
- Widget controls prioritize compactness over target size and accessibility.

## Positive Findings
- Strong foundational IA with clear separation of Main/Widget/Settings contexts.
- Robust recording/transcription flows with defensive handling in JS.
- Existing ARIA labels and keyboard navigation logic in settings tablist are good building blocks.

## Recommendations by Priority
1. Immediate
- Fix semantic control and target sizes in widget recording controls.
- Add global focus-visible system.
- Remove inline style mutations in JS status handling.

2. Short-term
- Normalize shared tokens for borders, states, focus, and buttons.
- Improve onboarding step-state clarity.
- Remove layout-property transitions from widget controls.

3. Medium-term
- Introduce refined motion and reduced-motion support.
- Align spacing/typography rhythm between Main and Settings.

4. Long-term
- Continue token migration to reduce hardcoded color usage in all windows.

## Suggested Commands for Fixes
- `harden`: accessibility targets, semantic controls, focus, reduced-motion.
- `normalize`: tokenization, consistent state styles, remove inline style attributes.
- `clarify`: copy simplification and action wording.
- `onboard`: first-run step-state clarity.
- `optimize`: transition/perf cleanup.
- `animate`: subtle motion choreography.
- `polish`: visual rhythm and consistency pass.
