## Design Context

### Users
Spanish-speaking professionals and learners who write in English daily. They use WhisloAI as a desktop overlay — translating text, dictating, and improving writing — without leaving their current app. Context: fast, interrupt-driven, always-on-top micro-interactions (translate selection, dictate, insert back). The widget must feel instant and lightweight, never like opening a separate app.

### Brand Personality
**Calm, precise, focused.** Three words that capture the tool's essence: it helps you write clearly without getting in the way. The interface should feel like a quiet, competent assistant — not a flashy AI product. Confidence comes from restraint, not decoration.

### Aesthetic Direction
**Visual tone:** Warm minimalism with organic green accents. Soft backgrounds with gentle radial gradients. Rounded surfaces (10–16px radius). Subtle depth through layered shadows, never harsh drops.

**Reference:** Grammarly — clean, helpful, non-intrusive writing tool. Specifically: the quiet confidence of Grammarly's desktop UI, the way it provides value without demanding attention.

**Anti-references:** No generic AI aesthetics. No purple gradients, no sparkle emojis, no "magical AI" vibes, no ChatGPT-clone patterns. WhisloAI is a productivity tool, not an AI playground.

**Theme:** Light mode only. No dark mode planned for now. The palette lives in a warm, airy, slightly green-tinted light space.

### Design Principles

1. **Invisible until needed.** The widget and anchor exist to disappear into the workflow. Every element earns its screen time. If it doesn't serve the immediate task (translate, dictate, insert), it doesn't belong.

2. **Calm feedback, never anxious.** Loading states use muted blue, success uses quiet green, errors are informative not alarming. Animations are gentle (120–220ms, `cubic-bezier(0.22, 1, 0.36, 1)`). Nothing pulses aggressively. The recording indicator is the only animated element that should draw attention, and even that should feel measured.

3. **Touch-friendly precision.** All interactive elements have a minimum 44px hit area. Buttons have gentle hover lifts (`translateY(-1px)`) but never bounce or wiggle. Rounded corners everywhere (10px for inputs/buttons, 12–16px for cards/panels, 999px for pills/badges).

4. **Consistent surface language.** Cards use `rgba(255, 255, 255, 0.94)` with subtle borders and gentle shadows. Selected/active states use the teal accent system (`#0f766e` family). Every surface follows the same border-radius, border-color, and shadow rules defined in the CSS custom properties.

5. **Accessible by default.** Focus-visible rings on all interactive elements. `aria-live` regions for status messages. `prefers-reduced-motion` media query disables all animations. Semantic HTML with proper roles. Color is never the sole indicator of state.

### Current Design System

**Colors (CSS custom properties in `src/styles.css`):**
- Background: `--bg: #f4f5f0` (warm off-white)
- Surface: `--surface: #ffffff`, `--surface-soft: #f8fbfa`, `--surface-elevated: rgba(255,255,255,0.94)`
- Ink: `--ink: #1f2a37`, `--ink-muted: #586173`
- Accent: `--accent: #0f766e` (teal), `--accent-strong: #0c645e`, `--accent-soft: #ecfbf6`
- Border: `--border: #dbe2ea`, `--border-soft: #c9dcd5`
- States: loading (blue), success (green), error (red) — each with fg, bg, border variants

**Typography:**
- Primary: `"IBM Plex Sans", "Avenir Next", "Segoe UI", sans-serif`
- Mono: `"IBM Plex Mono", "SFMono-Regular", ui-monospace, monospace`
- Labels: 12px, 700 weight
- Body: 14px
- H1: 28px
- H2: 17–22px (varies by context)

**Motion:**
- Easing: `cubic-bezier(0.22, 1, 0.36, 1)` (smooth deceleration)
- Fast: 120ms (hover, state changes)
- Medium: 220ms (panel enters, view transitions)
- Reduced motion: all animations and transitions disabled

**Spacing & Layout:**
- App max-width: 860px (main), 520px (compact), 1100px (settings)
- Padding: 12–20px
- Gaps: 4–14px
- Grid-based layouts throughout

**Component Patterns:**
- Buttons: 44px min-height, 10px border-radius, gentle hover lift
- Cards/Panels: 12–16px border-radius, subtle border + shadow, `rgba(255,255,255,0.94)` bg
- Pills/Badges: 999px border-radius
- Status indicators: pill-shaped with tone-based colors (neutral, loading, success, error)

**Tech Stack:**
- Tauri (Rust backend) + vanilla HTML/CSS/JS frontend
- No CSS framework — all custom CSS with custom properties
- Icons: Lucide (loaded via `lucide.min.js`)
- i18n: custom `data-i18n` attribute system

**Views:**
1. Main app (`index.html`) — compact translate/dictate view, 520px
2. Settings (`settings.html`) — full config with sidebar nav, 1100px
3. Widget (`widget.html`) — quick overlay toolbar, transparent bg
4. Anchor (`anchor.html`) — floating circular button overlay
